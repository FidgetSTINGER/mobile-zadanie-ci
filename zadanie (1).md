# Zadania z zajęć - Przegląd praktyk i narzędzi w programowaniu mobilnym

## Zadanie 1 – Analiza narzędzi developerskich

Wybrać narzędzie z poniższej listy i odpowiedzieć na pytania:
- do czego służy narzędzie?
- jakie funkcje oferuje, które ułatwiają development?
- w jakich sytuacjach jest niezbędne?
- przykłady praktycznego użycia w projektach mobilnych.

Narzędzia:
- Visual Studio Code
- Android Studio
- Expo DevTools
- Flutter DevTools
- React Native Debugger
- Flipper
- Xcode Simulator

## Zadanie 2 – Praktyka CI/CD

1.	Utworzyć prywatne repo na GitHubie.
2.	Dodać workflow CI dla jednej z technologii (szkielet):
    - lint
    - test
    - build
3.	Odpowiedzieć:
    - jaka część pipeline’u była najtrudniejsza i dlaczego?
    - co daje automatyzacja testów?
    - czym różni się pipeline aplikacji mobilnej od pipeline’u webowego?

⸻

## Zadanie 3 – Debugging aplikacji

Uruchomić aplikację mobilną (nawet prosty „Hello world”) i wykonać:
- logowanie wartości w konsoli,
- ustawienie breakpointów (VS Code lub Android Studio),
- prześledzenie działania funkcji krok po kroku,
- sprawdzenie działania “hot reload”.

Uzupełnić:
- zrzuty ekranu z konsoli/debuggera,
- opis błędu, który wykryli (może być sztuczny),
- opis, jak go naprawili.
